# LED 大屏动态背景墙资产命名规范

## Overview
爱奇艺数字资产标准 V2（2022/12）对 LED 大屏播放视频与配套工程素材的命名做了硬性约束。所有交付物的文件名 / 文件夹名都遵循同一根模式：

```
场景名称_序号_场景气氛时间_场景天气状态_<阶段标识>_<版本号>
```

「阶段标识」按交付物类型从 4 套词典里选；版本号格式恒为 `V01 / V02 / …`。

## When to use
- 给 LED 大屏播放视频成品命名（`*.mov / *.mp4` 等输出）
- 给 DCC 合成 / DCC 创作 / 虚幻引擎 / 调色四类工程文件夹命名
- 给摄影机原始拍摄素材文件夹命名
- 给制作中间产物（合成 / DCC / 调色的素材文件夹与序列帧）命名
- 整理一级 / 二级文件夹层级以便向爱奇艺数字资产库提交
- 需要查「中午 / 黄昏 / 阴天 / 实拍合成」等中文术语对应的英文缩写

**Don't use when**: 命名爱奇艺其它品类资产（电影、动漫、模型、IP 衍生品），或非爱奇艺 LED 项目 —— 它们各有自己的规范。

## Quick Reference — 五种命名方式

| 交付物 | 模式 | 样例（场景：理想之城 LXZC） |
|---|---|---|
| 最终视频成品 | `场景_序号_气氛时间_天气_制作方式_版本` | `LXZC_1_NT_N_RS_V01` |
| DCC / UE / 调色 工程文件夹 | `场景_序号_气氛时间_天气_制作工序_版本` | `LXZC_1_NT_N_CP_V01` |
| 摄影机原始素材文件夹 | `场景_序号_气氛时间_天气_M` | `LXZC_1_NT_N_M` |
| 制作中素材文件夹 | `场景_序号_气氛时间_天气_制作工序_版本` | `LXZC_1_NT_N_CP_V01` |
| 序列帧文件 | `场景_序号_气氛时间_天气_制作工序_版本_序列帧号.ext` | `LXZC_1_NT_N_CP_V01_00502.dpx` |

分隔符恒为半角下划线 `_`，扩展名只允许 `.dpx / .exr / .tiff`（序列帧）。

### 字段 1：场景名称
中文片名拼音首字母缩写、**大写**。例：`理想之城 → LXZC`。

### 字段 2：镜头序号
- 同一场景类型里的第几个镜头。仅一个镜头时 = `1`。
- **车拍镜头**特殊：由顺时针 5 块屏组成，前/右/后/左/上 = `F/R/B/L/T`，编号 `101-F、102-R、103-B、104-L、105-T`。

```
理想之城 / 镜头 1 / 夜晚 / 正常天气 / 全实拍 / 第一版 / 车拍
01 前：LXZC_101-F_NT_N_RS_V01
02 右：LXZC_102-R_NT_N_RS_V01
03 后：LXZC_103-B_NT_N_RS_V01
04 左：LXZC_104-L_NT_N_RS_V01
05 上：LXZC_105-T_NT_N_RS_V01
```

### 字段 3：场景气氛时间
| 中文 | 代码 |
|---|---|
| 日出前 | `BSR` — Before Sunrise |
| 日出 | `SR` — Sunrise |
| 早晨 | `MO` — Morning |
| 上午 | `FN` — Forenoon |
| 中午 | `Nn` — Noon ⚠ 原文首字母小写 |
| 下午 | `AN` — Afternoon |
| 黄昏 | `NF` — Nightfall |
| 日落 | `ST` — Sunset |
| 夜晚 | `NT` — Night |
| 深夜 | `LN` — Late_at_Night |
| 黄昏转夜晚 | `NF-NT` — Nightfall-to-Night |
| 夜晚转日出 | `NT-SR` — Night-to-Sunrise |
| 长时间延时拍摄 | `TL` — TimeLapse |
| 黄昏转夜晚延时拍摄 | `NF-NT-TL` — Nightfall-to-Night-Timelapse |
| 夜晚转日出延时拍摄 | `NT-SR-TL` — Night-to-Sunrise-Timelapse |

### 字段 4：场景天气状态
无特殊天气时一律用 `N`。

| 中文 | 代码 |
|---|---|
| 正常天气 | `N` — Normal |
| 下雨 | `R` — Rain |
| 下雪 | `S` — Snow |
| 阴天 | `CS` — Cloudy_Sky |
| 刮风 | `W` — Windy |
| 雾天 | `F` — Fog |

### 字段 5a：制作方式描述（用于最终视频成品）
| 中文 | 代码 |
|---|---|
| 全实拍场景 | `RS` — RealShot |
| 实拍 + CG 视效合成 | `RVC` — RealShot/VFX_Compositing |
| 全 CG 视效（视频类 / 实时虚拟制作类） | `VE` — VFX/Engine |

### 字段 5b：制作工序描述（用于工程与中间素材）
| 中文 | 工程代码 | 素材代码 |
|---|---|---|
| 合成 | `CP` — Compositing | `CPM` — Compositing_Material |
| DCC 创作 | `DC` — DCCsoft_Creation | `DCM` — DCCsoft_Creation_Material |
| 虚幻引擎 | `UE` — Unreal_Engine | — |
| 调色 | `CG` — Colorgrading | `CGM` — Colorgrading_Material |
| 摄影机原始 | — | `M` — Material |

⚠ 注意 `CG` 在本规范中 **专指 Colorgrading（调色）**，并非 Computer Graphics。CG 视效在最终视频成品里走字段 5a 的 `VE`。

### 字段 6：版本号
同一对象不同版本：`V01 / V02 / V03 / …`，恒为两位数，前缀 `V` 大写。

## 文件夹层级规范

一级文件夹 **= 视频内容名称（即字段 1 的场景中文名或对应英文，遵照项目约定）**。

二级文件夹按场景类型分三套：

### A. 全实景拍摄场景
```
<视频内容名称>/
├── LXZC_1_NT_N_M               # 摄影机实拍原始素材
├── LXZC_1_NT_N_CP_V01          # DCC 软件合成工程
├── LXZC_1_NT_N_CPM_V01         # DCC 软件素材（合成中间素材）
├── LXZC_1_NT_N_CG_V01          # 调色软件制作工程
└── LXZC_1_NT_N_RS_V01.mov      # 完成制作视频内容
```

### B. 实拍 + CG 视效合成场景
```
<视频内容名称>/
├── LXZC_1_NT_N_M               # 摄影机实拍原始素材
├── LXZC_1_NT_N_DC_V01          # DCC 内容制作工程
├── LXZC_1_NT_N_DCM_V01         # DCC 制作内容素材
├── LXZC_1_NT_N_CP_V01          # DCC 合成工程
├── LXZC_1_NT_N_CPM_V01         # DCC 合成素材
├── LXZC_1_NT_N_CG_V01          # 调色软件制作工程
└── LXZC_1_NT_N_RVC_V01.mov     # 完成制作视频内容
```

### C. 全 CG 视效（视频类 / 实时虚拟制作类）
```
<视频内容名称>/
├── LXZC_1_NT_N_DC_V01          # DCC 内容制作工程
├── LXZC_1_NT_N_DCM_V01         # DCC 制作内容素材
├── LXZC_1_NT_N_CG_V01          # 调色软件制作工程
└── LXZC_1_NT_N_VE_V01.mov      # 完成制作视频内容
# —— 或 ——
├── LXZC_1_NT_N_UE_V01/         # 虚幻引擎制作工程（含相关制作素材、操作蓝图与插件）
├── LXZC_1_NT_N_CG_V01
└── LXZC_1_NT_N_VE_V01.mov
```

## Common Mistakes

| 错误 | 纠正 |
|---|---|
| 用驼峰 / 中划线分隔字段 | 通用规范：字段之间一律 `_`；字段内部允许 `-`（如 `101-F`, `NF-NT-TL`） |
| 父级目录或文件名里**带半角空格** | **本项目特例**：项目前缀 ⇄ 场景名之间**无任何分隔符**，拼音首字母直接连写；父级目录中文名同样去掉空格 —— 见下方「项目级约定」 |
| 忘了 `V01` 直接 `_v1` 或省略版本 | 版本号位是**必填**，恒两位数，大写 `V` |
| 把 CG 视效写成 `CG` | 字段 5a 用 `VE`；`CG` 只代表「调色 Colorgrading」 |
| 中午写成 `N-Noon` | 原规范是 `Nn-Noon`（首字母小写 n + 第二位小写 n），按原文照抄 |
| 序列帧扩展名用 `.png / .jpg` | 序列帧只用 `.dpx / .exr / .tiff`（成品截图才用 jpg） |
| 摄影机原始素材文件夹补版本号 | 摄影机原始素材文件夹**不带版本号**，模式只到 `_M`；且**不可改原始拍摄文件名** |
| 制作中素材文件夹复用工程文件夹名（都是 `..._CP_V01`） | 原规范允许同名 —— 通过**所在二级目录**区分。如果想避免歧义，可把制作中素材文件夹替换为 `CPM / DCM / CGM` 变体并和团队约定 |
| 车拍五块屏只编 `1/2/3/4/5` | 车拍必须用 `101-F … 105-T`，组合方向是**顺时针** |
| 「正常天气」直接省略 | 没特殊天气也要写 `N`，字段不能空 |

## 项目级约定 (Project Override)

⚠ **本节盖过**「Quick Reference」+「Common Mistakes」中的对应条目，**仅适用于本项目**。其它项目按 iQiyi 通用规范走，不要外推。

### 父级目录与项目前缀 — **去空格、直接拼接**

- 父级目录中文名**取消半角空格**：`常州 戚墅街道/` → `常州戚墅街道/`
- 文件名项目前缀 ⇄ 场景名之间**取消任何分隔符**，拼音首字母**直接连写**：`CZ` + `QSJD` → `CZQSJD`
- 与父级目录**镜像对齐**（不是镜像空格，是镜像「项目+场景」这个 1:1 结构）

| 父级文件夹 | 场景（→ 场景名） | 项目（→ 前缀） | 子文件名（最终视频成品） |
|---|---|---|---|
| `常州戚墅街道/` | 戚墅街道 → `QSJD` | 常州 → `CZ` | `CZQSJD_101-F_FN_N_RS_V01.mov` |

### 规则细则

- **项目前缀 ⇄ 场景名**：**无分隔符**（拼音首字母直接连写 `CZQSJD`）
- **场景名 ⇄ 镜头序号**：下划线 `_`
- **镜头序号 ⇄ 气氛时间 ⇄ 天气 ⇄ 制作方式 ⇄ 版本**：全部下划线 `_`
- **天气**字段必须出现，无特殊天气用 `N`
- **天气 ⇄ 制作方式**两个字段必须拆开，例 `N_RS`（不要黏成 `NRS`）
- **禁止空格**：父级目录、子文件名、字段内部都**不允许出现半角空格**；连字符 `-` 只在字段内部使用（`101-F`、`NF-NT-TL`）

### 修正对照

```
❌  CZ QSJD_101-F_FN_N_RS_V01.mov    错：项目与场景之间有空格（已废止）
❌  CZ_QSJD_101-F_FN_NRS_V01.mov    错：用了下划线；天气与制作方式黏合
❌  QSJD_101-F_FN_N_RS_V01.mov      错：砍掉项目前缀，丢失项目层信息
✅  CZQSJD_101-F_FN_N_RS_V01.mov    对：项目+场景直接拼接，天气与制作方式拆分
```

### 仅限本项目

若换到 iQiyi 直接交付（无项目层目录），回到通用规范 `QSJD_101-F_FN_N_RS_V01.mov`（砍 `CZ` 前缀；保留 `N_RS` 拆分）。

## 制作内容截图规则（命名相关补充）
- 截图与视频成品同名（多张则后缀 `-01 / -02 / -03`），与视频放在同一文件夹。
- 格式：JPEG、8bit、RGB 采样；推荐 DaVinci Resolve / Scratch 导出全尺寸。
- **单一场景**：起始 1 张。**转换场景（日夜 / 夜日 / 延时）**：起始 / 过程 / 结束 共 3 张。

## 提交时的检查项
- 视频内容硬性要求：分辨率 ≥ 4K-UHD（3840×2160）；高质量编码（ProRes422HQ / ProRes4444 / RAW）；**禁用 MP4 / MKV**。
- HDR 项目：输出 HDR10 ProRes422 / ProRes422HQ。
- 文件齐：本规范第 4 节列的素材 / 工程 / 截图缺一不可。
- 一并提交 **《爱奇艺数字资产 LED 大屏播放素材统计表》** 模板，杀青前一周交 V1.0、杀青后两周交完整资产。
